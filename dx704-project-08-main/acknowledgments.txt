### Class Explanation: `RoverSimulator`

This class models a simple **Mobile Rover** operating on a 2D square grid. It defines the rules for movement, state, and rewards, essentially creating a small simulated world for an agent to learn in.

1.  **State Representation:**
    * The rover's state is defined by three components: its grid coordinates `(x, y)` and its current `direction` (an integer from 0 to 7).
    * The grid size is determined by `resolution`.
    * The methods `construct_state` and `decode_state` are crucial; they convert the human-readable `(x, y, direction)` tuple into a single, unique integer ID (the state ID) and back again.

2.  **Movement and Directions:**
    * The `DIRECTIONS` array defines the eight possible 45-degree headings (North, Northeast, East, etc.) using `(dx, dy)` vectors.
    * The rover has three actions:
        * `-1`: Turn 45 degrees left (counter-clockwise).
        * `0`: Maintain current direction.
        * `1`: Turn 45 degrees right (clockwise).
    * In every step, the rover **always moves one unit** in the direction it is currently facing, and *then* the direction is updated based on the chosen action. Movement is constrained by the grid boundaries (clamped to stay within $[0, \text{resolution}-1]$).

3.  **Goal and Rewards:**
    * The **Terminal State** (the goal) is hardcoded to be the exact center of the grid, specifically at `(resolution // 2, resolution // 2)` with direction 0.
    * The reward is simple: `1` if the rover lands exactly on the terminal state, and `0` otherwise.

4.  **Simulation Setup:**
    * The constructor pre-calculates a list of legal `initial_states` from which a simulation can begin. These starting points are chosen from the corners and center lines of the grid, but explicitly exclude the terminal state itself.
